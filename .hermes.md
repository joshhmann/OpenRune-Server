## Strict workflow

- One active milestone at a time.
- Red build freezes new content. Only repair, review, or documentation tasks may proceed.
- No implementation task may be scoped from memory alone. Attach a research snapshot from corpus/raw-cache/osrs-mcp evidence first.
- Runtime-required tasks cannot close silently. If no client/bot walkthrough occurred, mark `runtime-test: required` and keep the milestone open.
- No task is done because a worker says done. It is done when compile/build/runtime/doc evidence is attached.
- Every close updates `docs/worklog-239.md` and relevant status/deferred docs before opening the next card.
